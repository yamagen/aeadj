# 次のスレッドへの申し送りです。

このスレッドでは、AEAD.json の見直しを、主に No.238 から No.330 まで進めた。作業の中心は、既存項目に `date-history`、`variants-ja/en`、`intent-ja/en`、`tags-ja/en`、`adjusted-expression-ja/en` を補い、英語表現、roman-ja、notes-ja/en の明らかな誤りや不自然さを最小限修正することだった。基本方針は、元の観察や語りの面白さをなるべく保持し、AIが勝手に一般化したり教育的に要約しすぎたりしないことである。

今回、特に重要な方針変更として、`adjusted-expression-ja/en` は「説明文」ではなく、「同じ場面で実際に言える調整された発話」として作ることを確認した。たとえば No.285「きっと大丈夫」では、即時表現が `A「大丈夫かな？」B「きっと大丈夫だよ」` であるのに対し、調整表現は `A「大丈夫でしょうか？」B「ええ、おそらく、大丈夫じゃないでしょうか」` のような、同じ場面で時間をかけて丁寧に言い換えた発話になる。この方針は No.286 以降の見直しで適用されている。今後も `adjusted-expression` は「相手を安心させている」「理由を尋ねている」のような意味説明ではなく、実際の発話として出す。

作業中の表記方針として、日本語の引用符は「」を使い、英語例では基本的に A: ... B: ... の形を保つ。日本語の例文では、ルビ用の `(漢字:よみ)` 形式を維持する。ただし、`(うん)` や `(ああ)` のようにルビではない括弧は通常表記に直した。`roman-ja` は必要に応じて分かち書きし、`ha` は助詞「は」の発音として `wa` にする方針をとった。ただし、既存の `index` や `img-sub` は、ユーザーが元のまま保持したい場合があるので、勝手に変更しすぎない。No.289 では、一度 `index` を `tanakasantekiniwadoudesuka` に変更したが、ユーザーが元の `santekiniwadoudesuka` を再提示したため、最終的には元の `index` と `img-sub` を保持した。

No.324「詳しくは、この曲のあと」では、すでに `tags-ja/en` と `adjusted-expression-ja/en` が入っていた。ここでは `date-history`、`variants`、`intent` を補い、`tags` の粒度を調整した。`adjusted-expression` はすでに発話型だったので、その方針を保持し、ルビを加えた。`notes` では「述語が省略されても、その場の状況から補える」という即時文法の特徴を明確にした。

No.323「いいですね、すごく」では、話している途中でまず `いいですね` と評価を出し、その後から `すごく` を付け足して評価を強める形として整理した。これは書きことばでは `すごくいいですね` の語順になるが、会話では後置された `すごく` が自然であり、即時文法らしい例として扱った。

No.300「今、思い出した！」では、過去形が過去の出来事ではなく、現在の気づきや再認識を表すことを notes に保持した。また `会議があるんだった` の `だった` も未来の予定について、今思い出したことを示す過去形として説明した。

No.309「誰を待っているの？」では、`の` は所有ではなく、やわらかく尋ねる終助詞として整理した。また `誰を` の `を` は、待っている対象を明示し、待っている主体と区別するためのものと説明した。カジュアル体で文末に `か` を使うと乱暴に響く場合があり、カジュアルでは `の`、フォーマルでは `ですか` / `ますか` が適切という観察を保持した。

No.330「どうしてだめなの？」では、`な` は単なる形容詞の印ではなく、ナ形容詞 `だめ` と文末の `の` をつなぐ形として説明した。また、文末の `の` は理由を述べるのではなく、相手に理由や事情の説明を求める働きとして整理した。`title / expression-en` は自然な `Why not?` に整えた。

No.327「アレルギーがあります」では、`がある` を単純な subject marker としてではなく、「存在するもの、持っているもの」を示す構造として説明した。No.328「つながらない」では、電話だけでなくネット接続にも使えること、`切れてる` は `切れている` の縮約形で結果状態を表すことを整理した。No.329「どうかしたの？」では、`の` は理由を述べているのではなく、相手に説明を求める終助詞として整理した。

No.291「多くの人」は、`The majority of people` では強すぎるため、`Many people` に整えた。`多くの人` は必ずしも「大多数」を意味するとは限らない。No.297「乗りかかった船」は、英語の `be in the same boat` とは異なり、「関与した以上、途中でやめられない」「最後までやるしかない」という個人の責任を強調する表現として整理した。No.317「ちなみに」は、`on the related note` を自然な `on a related note` にしつつ、例文では `By the way` にした。No.320「今回、特別ね」は、単なる `special` ではなく「今回だけの例外的許可」として `Just this once` / `This time, it's an exception` に寄せた。

全体として、今後の見直しでも、各項目に対して次の作業を続ける。`date-history` には元の日付を数値化したものと見直し日を入れる。`variants-ja/en`、`intent-ja/en`、`tags-ja/en` を補う。`adjusted-expression-ja/en` は必ず同じ場面で言える調整された発話にする。`notes-ja/en` は元の観察を尊重し、誤りや文法説明の不正確さだけ最小限直す。英語が不自然な場合は自然な英語に整えるが、元の日本語表現の即時性や言い淀み、後置、冗談、皮肉、含みは消さない。
